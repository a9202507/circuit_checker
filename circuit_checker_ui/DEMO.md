# Circuit Checker Rules UI - 功能演示

## 系統架構

```
┌─────────────────────────────────────────────────────────────┐
│                  前端（Vite + React + TS）                   │
│                                                               │
│  ┌─────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │ YAML 編輯器  │  │ 規則構建器    │  │ 新手指南      │         │
│  └─────────────┘  └──────────────┘  └──────────────┘         │
│                                                               │
└─────────────────────────────────────────────────────────────┘
                            ↓ HTTP / JSON
┌─────────────────────────────────────────────────────────────┐
│           後端（FastAPI + Python + PyYAML）                   │
│                                                               │
│  POST /validate      ← 驗證 YAML 語法                        │
│  POST /parse         ← 解析上傳的 YAML 檔案                  │
│  POST /generate      ← 生成 YAML 內容                        │
│  GET  /templates     ← 取得規則範本                          │
│  GET  /examples      ← 取得完整範例                          │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

#### YAML 編輯器功能演示

**功能 1：驗證 YAML**
```
使用者貼上或編輯 YAML 內容
    ↓
點擊「✓ 驗證」
    ↓
後端驗證 YAML 語法和規則內容
    ↓
返回驗證結果
    ├─ 如果有效：顯示規則統計
    └─ 如果有錯：顯示詳細錯誤信息（欄位、行號、建議修改）
```

**功能 2：匯入 YAML 檔案**
```
點擊「📤 匯入 YAML」
    ↓
選擇本地 .yaml / .yml 檔案
    ↓
後端解析檔案
    ↓
自動驗證
    ├─ 成功：顯示內容和規則統計
    └─ 失敗：顯示錯誤信息
```

**功能 3：匯出 YAML**
```
驗證通過後，點擊「📥 匯出 YAML」
    ↓
瀏覽器下載 rules.yaml 檔案
    ↓
可用於 circuit-checker 執行檢查
```

**功能 4：載入範例**
```
點擊「📋 載入範例」
    ↓
自動載入完整的規則範例
    ↓
包含 4 種規則類型各一個
    ↓
點擊「驗證」查看完整規則集
```

### 場景 4：錯誤處理和驗證反饋

**驗證失敗示例**
```
使用者上傳無效 YAML：
---
rules:
  - name: "Bad Rule"
    type: "invalid_type"      ← 錯誤：無效規則類型
    net:                       ← 錯誤：value 欄位缺失

系統反應：
✗ 驗證失敗
找到 2 個錯誤

[rule_0] 無效的規則類型 'invalid_type'。有效值：
         net_cap_to_gnd, pin_floating, pin_to_pin_resistor, pin_cap_to_gnd

[value]  net_cap_to_gnd 需要 'value' 欄位
```

## API 請求/響應示例

### 1. 驗證 YAML

**請求：**
```bash
curl -X POST http://localhost:8000/api/validate \
  -H "Content-Type: application/json" \
  -d '{
    "content": "rules:\n  - name: \"Test\"\n    type: \"net_cap_to_gnd\"\n    net: \"Vcc\"\n    value: \"0.1uF\""
  }'
```

**響應 (成功)：**
```json
{
  "valid": true,
  "rules_count": 1,
  "rules": [
    {
      "name": "Test",
      "type": "net_cap_to_gnd",
      "net": "Vcc",
      "value": "0.1uF"
    }
  ],
  "errors": []
}
```

**響應 (失敗)：**
```json
{
  "valid": false,
  "rules_count": 0,
  "rules": [],
  "errors": [
    {
      "field": "type",
      "message": "Rule 0: 無效的規則類型 'bad_type'",
      "line": null
    }
  ]
}
```

### 2. 上傳 YAML 檔案

**請求：**
```bash
curl -X POST http://localhost:8000/api/parse \
  -F "file=@rules.yaml"
```

**響應：**
```json
{
  "filename": "rules.yaml",
  "valid": true,
  "rules_count": 4,
  "rules": [...],
  "errors": []
}
```

### 3. 生成 YAML

**請求：**
```bash
curl -X POST http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "Vcc cap",
      "type": "net_cap_to_gnd",
      "net": "Vcc",
      "value": "0.1uF",
      "tolerance": "10%"
    }
  ]'
```

**響應：**
```json
{
  "valid": true,
  "yaml": "rules:\n- name: Vcc cap\n  type: net_cap_to_gnd\n  net: Vcc\n  value: 0.1uF\n  tolerance: 10%\n",
  "filename": "rules_20240314_120530.yaml"
}
```

## 實際使用流程

### 完整工作流：從設計到驗證

```
Step 1: 電路設計師完成原理圖
    ↓
Step 2: 打開 Circuit Checker Rules UI
    ↓
Step 3: 點擊「📚 新手指南」了解規則類型
    ↓
Step 4: 切換到「🔨 規則構建器」
    ↓
Step 5: 根據設計添加檢查規則
    ├─ 電源去耦電容（0.1uF）
    ├─ 振盪器晶體負載電阻
    ├─ 信號完整性電阻終端
    └─ 浮空 pin 檢查
    ↓
Step 6: 生成 YAML
    ↓
Step 7: 驗證語法（自動）
    ↓
Step 8: 匯出 rules.yaml
    ↓
Step 9: 在 CI/CD 中使用
    $ circuit-checker --netlist ./netlist --rules rules.yaml
    ↓
Step 10: 檢查結果
```

## 開發者參考

### 新增自訂規則類型

1. **更新後端驗證**：編輯 `api/main.py` 的 `validate_rule()` 函數
2. **更新前端表單**：在 `RuleForm.tsx` 中新增欄位
3. **更新範本**：在 `/templates` 端點中新增新類型

### 自訂 UI 主題

編輯 `tailwind.config.js`：
```javascript
theme: {
  extend: {
    colors: {
      pass: '#16a34a',      // 通過色
      fail: '#dc2626',      // 失敗色
      // ... 自訂其他顏色
    }
  }
}
```

## 性能指標

- API 驗證響應：<100ms
